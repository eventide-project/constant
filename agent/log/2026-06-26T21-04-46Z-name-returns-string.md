# Design correction: Constant#name returns a String (mirroring Module#name, the type it is derived from), not a Symbol. Ruby is mixed on this — Module#name is a String, Module#constants/Method#name are Symbols — and the strongest specific precedent for a wrapped module/class's own name is Module#name (a String). #constant_names stays Symbols (mirroring Module#constants); the only discipline this asks is routing name-existence checks through const_defined? (type-agnostic), not comparing #name against the Symbol list. Surfaced as a GATE-1 correction during the Name experiment
