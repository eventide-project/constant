# Terminology: "mediates for" is the canonical verb for the Constant↔raw-constant relation — a Constant *mediates for* a raw Ruby constant (the accessor standing between efferent code and the bare constant, fronting name/namespace/equality). Retires "wrap/wrapper" and "holds" as the relation verb. The equality outcome contexts were named with it ("Equal/Eql?/Equal hash/Unequal when mediating for the same/different constant(s)"); the one exception is non_constant.rb, named "Unequal when the other is not a Constant" — there the test establishes the type guard, not the relation, so faithful-to-the-assertion wins over one-verb-everywhere. Distinguished by slot during the dialogue: "represents/holds" name the *being* (identity/referent) relation, "mediates for" names the *doing* (functional/accessor) role; the human fixed "mediates for" as canonical and "for the same constant" carries the identity criterion in the equality names.
