# Test structure: a `test` block contains ONLY the assertion; never inline a method call in an assertion — assign every operand (the actuation result and the compared-against value) to an explaining variable declared in the enclosing `context` block; encase the test block in a context where those variables live. The assertion-specific, stronger form of the no-inline-method-call-arguments rule, and it extends the first-turn "assert an explaining variable" to *both* operands (e.g. `other_constant = Constant.new(control_constant); test { assert(constant == other_constant) }`, not `assert(constant == Constant.new(...))`).
