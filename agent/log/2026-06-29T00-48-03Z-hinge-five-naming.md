# Process: the hinge cycle gains a **fifth hinge — naming**. Naming the outcome `context` (the test's name) is a design hinge in its own right, not a loose closing step — a name that mis-describes what the assertion establishes is a design error — so it gets the hinge mechanism (options-or-accept + mandatory "Chat about this" escape). It stays **deferred to the feature's close** (per the first-turn rule, outcomes develop unnamed); at the close each outcome's name is settled via options (the one-outcome-at-a-time naming rule). For a single-outcome feature, naming follows the implementation hinge directly. Cycle is now actuation → assertion → controls → implementation → naming.
