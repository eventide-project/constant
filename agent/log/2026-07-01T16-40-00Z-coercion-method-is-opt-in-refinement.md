# Settled: the `Constant()` coercion method is an opt-in Kernel refinement (`Constant::Coerce`), not a global Kernel method or an Object singleton method. Ruby's coercion idiom (`Integer()`/`String()`) is a private Kernel instance method, so a bareword `Constant(x)` requires a Kernel-level method; a gem planting one unconditionally pollutes every process, against the library's ethos. A refinement is the polite library form — consumers activate it per-file with `using Constant::Coerce`. `Constant(x)` is a method call (parens disambiguate it from the `Constant` module constant), an **idempotent** front door: if the value is already a `Constant` it is returned unchanged, otherwise it delegates to `Constant.build`. Idempotency lives in the coercion method — that is its whole point; `build` stays a strict factory and is left untouched. To be built test-first
