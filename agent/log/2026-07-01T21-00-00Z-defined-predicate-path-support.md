# Investigated and fixed `Constant.defined?` path support. Investigation: native `const_defined?` already handled defined/undefined `::`-paths correctly (returns booleans, never raises) — but **leaked a `TypeError`** when a path traversed *through* a literal (a non-module mid-path segment, e.g. `defined?("SomeLiteral::x")`), violating the predicate's never-raising contract. Fix (test-first, routed through resolution): `Constant.defined?` now does `get(namespace).get(name, inherit:); true rescue Constant::Error => false` — `#get` returns a Constant for any defined name (module or literal) and raises `Constant::Error` for an undefined name *or* a path through a literal, so both collapse to `false`. Behavior-neutral for every existing `defined?` test; closes the `TypeError` leak; and the method-level rescue makes `defined?` robustly never-raising (an undefined *namespace* name now also returns `false` rather than raising). Supersedes the "`Constant.defined?` path support out of scope" aside in `2026-07-01T19-40-00Z-nested-path-strings-built.md`.
