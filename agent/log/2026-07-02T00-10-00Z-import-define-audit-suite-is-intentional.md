# Closed the import-define-test-convention-audit as mis-premised: the older `Import` / `Define` test suite is **intentional**, not convention-drift. Audited against the `constant/` suite's conventions and found each apparent divergence is a deliberate, coherent choice: (1) **unnamed tests** — the context names the outcome and carries the local instrumentation; the `test` block is the bare assertion (a valid alternative to the name-the-test form); (2) the **`control_` / `"Control "` two-tier scheme** — `control_` marks the deterministic fixture *values*, while the stateful domain entities a side-effecting operation acts on are named by domain role (`origin_constant`, `destination_constant`, `alias_constant`), which a blanket `control_` would flatten; the `"Control "` comment prefix tracks the same split; (3) **instrumentation-only fetches** (e.g. `imported_constant` in the "returned" context) — deliberate, for the comment; (4) the **`assert_raises` message check** on `NameError` — intentional: `NameError` is a generic Ruby error, so the exact-message assertion disambiguates it from an unrelated `NameError` in the execution path, whereas the library-specific error classes (`Constant::Error`, `Constant::Import::Error`, `TypeError`) are self-evidently the right failure and need no message. So the audit's premise (conform the suite to `constant/`) does not hold. The examination *did* drive two deliberate improvements (choices, not conformance): the API vocabulary rename source/receiver → **origin/destination** with the superfluous `target_constant` test indirection dropped (commit 09f16fb), and in the resolution contexts, `eval` → `Object.const_get` (b5af27e) plus rewording "Imported constant resolution via destination" → "Imported constants are resolvable via destination" with the variable renamed to `imported_constant` (ab3275c). Suite verified (75). Resolves and removes the import-define-test-convention-audit deferred item
