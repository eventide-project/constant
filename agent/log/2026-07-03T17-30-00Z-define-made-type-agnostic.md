# Made `Constant::Define` type-agnostic rather than pinning it as module-only — reconsidering (not merely closing) Tier 1 item 3 of the coverage-discrimination-gaps deferred item. The item observed that `define_constant.rb` only asserted the name was defined and that the return `== const_get`, so a defective `Define` binding a non-module (e.g. a String) would still pass; it proposed asserting the result **is a module**. Examining that surfaced a prior question: is `Define` meant to be module-only, or is the *kind* of constant incidental? The implementation had silently committed to modules (hard-coded `::Module.new`, no value parameter) — on track *only* under the module-factory reading, which is the reading the README stated and `Constant::Import` relies on (Import calls `Define.(alias, destination)` and then `const_set`s imported constants into the result, so the alias target must be a module). Decision: make `Define` **type-agnostic** while keeping the module as the default. Signature becomes `call(constant_name, destination_constant, constant_value=nil)`; a `nil`/omitted value coalesces to a fresh `::Module.new` in the body (per the optional-params-default-in-body convention), so existing two-argument callers — `Import` and the module-default test — are unchanged, while a supplied value is assigned as-is, letting a literal be defined. (`nil` as a *defined value* is unsupported — `nil` reads as "omitted"; a sentinel was considered and declined as not worth the surface for an exotic case.) The actuation shape settled on an optional trailing **positional** value (over a keyword, or a required value that would have edited Import). Designed test-first: the literal test went red against the two-argument implementation (an arity error), then green once the value parameter was added — a **contained** red/green (the module-default test stayed green throughout), per the first-implementation-red/green rule. The flat `define_constant.rb` was reorganized into `define_constant/{module,literal}.rb`: `module.rb` keeps the defined / returned assertions and **adds** `instance_of?(::Module)` (closing the original gap — a non-module return now fails), and `literal.rb` asserts the given value is returned. The README's Define section now documents the `constant_value` parameter (with a literal example and an API-table row). Commits: impl + test reorg `9330cc6`, README `c18a4fb`. Suite verified (87 tests). Resolves Tier 1 item 3; item 2 (error-message assertions) and Tiers 2–3 of the coverage-discrimination-gaps item remain.
