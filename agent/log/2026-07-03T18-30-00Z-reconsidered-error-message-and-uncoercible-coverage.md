# Reconsidered two coverage-discrimination-gap items (Tier 1 item 2, Tier 2 item 5) and found neither was the coverage task it looked like. **Item 2 (error messages):** the premise was "all 8 `assert_raises` check only the class; add message assertions." Scrutiny — *are message assertions necessary to determine the correct error was raised under the expected conditions?* — showed they are not: the class plus the test's condition already determine that, **except** in one situation, which became a rule: assert message content only when the **same error class is reachable from multiple sites in one execution path**, so the class alone cannot confirm which site fired. That is the two `::`-path cases — `nested_undefined` (`Constant::Error` at the head segment vs the tail) and `nested_into_literal` (the module "not defined" site vs the literal "primitive value" `#get`) — where the message is the sole discriminator; message checks kept there (commit `e8c651f`). The single-site coercion `TypeError` and `Literal#get` errors keep class-only assertions. Rule recorded: `assert-error-message-only-as-sole-discriminator`. **Item 5 (non-nil un-coercible):** `Constant(42)` is behaviorally identical to `Constant(nil)` — same `else` branch — and its only distinct effect is the message's class-name arm, a single-site `TypeError` not worth asserting per the same rule; so that half dissolved (no test). The `Constant(:sym)` half instead surfaced a **defect**: `Constant()` accepted only Module/String, so `Constant(:sym, ns)` raised `TypeError` though `Constant.get(:sym, ns)` resolves it and the coercion is documented as a front door over `get`. Fixed test-first — Symbol added to the coercion's name guard, and `coerce/namespace_name.rb` reorganized into `namespace_name/{string,symbol}.rb` — commit `83e92be`. Suite verified (88 tests). Resolves Tier 1 item 2 and Tier 2 item 5; item 4 (direct Symbol inputs for `get` / `defined?` / `Define`) is partially advanced (the coercion now exercises `get` with a Symbol transitively) but its direct cases remain; items 6, 7, 8 remain.
