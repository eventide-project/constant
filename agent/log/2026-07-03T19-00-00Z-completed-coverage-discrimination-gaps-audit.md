# Completed the coverage-discrimination-gaps audit — all 8 items resolved, several by reconsideration rather than by becoming the tests they first looked like. **Tier 1:** (1) `inherit` was completely untested — added `When inherit is true`/`false` discrimination pairs via a new `ancestor:` control on `Controls::Constant.example`, covering `Constant.get`, `Constant::Module#constants`, `#constant_names`, class-level `Constant.defined?`, and instance `#defined?`; green-on-arrival. (2) Error messages — reconsidered: message content is worth asserting only where the *same error class is reachable from multiple sites in one execution path* (→ the `assert-error-message-only-as-sole-discriminator` rule), so it reduced from "8 assert_raises" to the two `::`-path cases (`nested_undefined`, `nested_into_literal`); the single-site coercion and `Literal#get` errors keep class-only assertions. (3) `Define` "need not create a module" — reconsidered into a design change: made `Define` **type-agnostic** (optional `constant_value`, default a new module, preserving the `Import` alias-target path) so a literal can be defined; the module-default path now asserts `instance_of?(::Module)`. **Tier 2:** (4) Symbol name inputs — found the *only* Symbol-rejecting guard in the library was the coercion's; `get`/`defined?`/`Define` otherwise normalize via `name.to_s` or hand names to `const_set`, so Symbols were covered for `get` directly (coerce symbol test), `defined?` transitively (shared `Module#get` path), and `Define` directly. (5) Non-nil un-coercible values — `Constant(42)` is behaviorally identical to `Constant(nil)` (same `else` branch), so it dissolved; the `Constant(:sym)` half instead surfaced a **defect** — the coercion rejected Symbol names that `Constant.get` accepts — which was **fixed** test-first (commit `83e92be`). (6) `Constant` as namespace — added the missing case exercising the `namespace.is_a?(Constant)` branch of `Constant.get`. **Tier 3:** (7) 3+-segment paths — added an `A::B::C` case forcing two recursion steps, so a recursion that only splits one `::` would be caught. (8) `Define` redefinition — observed `Define` behaves **exactly** as Ruby's `const_set` (overwrite + "already initialized constant" warning, proven identical to a bare `const_set` and to literal `X = 1; X = 2` reassignment); decided to keep it transparent to Ruby, so there is no library redefinition *policy* to protect — a "redefinition replaces" test would test Ruby's `const_set`, not the library, so none was written (→ the `do-not-test-the-platform` rule). Outcomes: two rules emerged (sole-discriminator; don't-test-the-platform), one defect fixed (Symbol coercion), one design change (type-agnostic `Define`); the suite grew from 75 to 92 tests, all verified. Lesson: an audit premise is a hypothesis — under scrutiny, three of the eight items (2, 3, 5, and arguably 8) were not the coverage tasks they first appeared to be.
