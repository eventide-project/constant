# Made the name accessors conform to Ruby for anonymous modules (review finding #8): `Constant::Module#name`, `#namespace`, and `#full_name` now return `nil` for a module whose `Module#name` is `nil` (an anonymous `Module.new`), instead of `#name`/`#namespace` raising a raw `NoMethodError` from `nil.rpartition` while `#full_name` alone returned `nil`. The developer directed the design: since `Module#name` is itself a total query returning `nil` for anonymous modules, the library's name-derivation should mirror that rather than fight the platform. Driven test-first through the hinge cycle — actuation/assertion settled by the directive; controls = inline `Module.new` (an anonymous module is the primitive itself, no seeding); `#name`/`#namespace` were the two red→green driven outcomes (contained red: `nil.rpartition` aborted only the two new tests), `#full_name` was already `nil` and its `nil` was locked as a deliberate green-on-arrival regression guard (developer's call, an intentional exception to the drop-green-on-arrival default). Guard placed in the `Constant.name`/`Constant.namespace` class helpers (where the core derivation and the `value.name` dereference live), keeping them total functions and the instance methods thin delegators — chosen over guarding the instance methods, which would leave the class helpers partial and crash-prone if called directly. Naming settled per accessor for the outcome each asserts: "Has no name when anonymous" (`#name`), "Has no namespace when anonymous" (`#namespace`), "Has no full name when anonymous" (`#full_name`). `full_name.rb` promoted to a `full_name/` directory (`nested.rb` + `anonymous.rb`), mirroring `name/` and `namespace/`. Suite 92 → 95, all verified. Amended the string-outputs rule to record the `nil`-conformance exception. Closes review finding #8.
