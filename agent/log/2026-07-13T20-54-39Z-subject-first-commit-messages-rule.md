# Added a subject-first commit-message rule (`agent/rules/methodology/2026-07-13T20-54-39Z-subject-first-commit-messages.md`): commit subjects lead with the changed thing in passive voice / indicative mood ("Widget reconciliation is corrected", not "Fix …"), omitting the verb for new items, using "is corrected"/"is clarified", and dropping the 50-char subject limit — per Aaron Jensen's guide (https://github.com/aaronjensen/software-development/blob/master/commit-messages.md). Placed in `methodology/` per the developer's instruction (the other commit rules live in `git/`; the new rule cross-references them). Consistent with the existing commit-says-task-is-done-not-mark-done rule.
