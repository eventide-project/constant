## 2026-06-20 — Session: establishing TDD as a design tool, then formalizing the gates

### Framing: TDD is a design tool, not a ceremony
TDD here is a design tool; red-green-refactor is choreography that only ever existed to insert a human's attention. A test written before its implementation is trivially going to fail — that failure teaches nothing, so we don't stop to observe the red bar. The point is the **cradle** (refined across the dialogue: jig → harness → *cradle* — it holds the implementation in proper position without gripping it rigid, supporting generation within the dictates of the efferent view while leaving it free to move inside those bounds). The target is **solubility**: how readily a unit dissolves into use. "Efferent in, efferent out" — condition on this project's high-solubility code, not the average of code seen. Design judgment stays with the human. → rule `…18-07-21Z-tdd-as-design-tool.md`.

### Terminology: efferent, never caller
The use-site perspective is always **efferent** (efferent side / reference / view), never "caller." "Actuation" and "the call to the unit under test" stay fine — they name the invocation, not the perspective. Applied retroactively across the TDD rules. → rule `…18-30-05Z-tdd-efferent-not-caller.md`.

### The process, turn by turn
- **Turn one:** start a feature by writing *only* the test file — proper context nesting, one `test` block, one assertion against an **explaining variable** holding the result of actuating the concern. No implementation; don't pause on the inevitable failure. → rule `…18-11-57Z-tdd-first-turn-test-file.md`.
- **Next turn (described, not yet its own rule):** bring the unit into existence inside the cradle, generate straight through toward solubility, run the test to *confirm* (verification, not the green half of a ceremony).
- **Then loop by outcome;** exit on solubility, not coverage.

### Test structure (exemplar: `define_constant.rb`)
Actuate once at the top of the feature context; assert each outcome in its own inner context. Explaining variables carry the assertion (never assert an inline expression); narration is layered (`comment` scenario-wide, `detail` context-local); one assertion per `test`, one outcome per `context`. → rule `…18-13-51Z-tdd-test-structure.md`.

### Human-in-the-loop: gate on the call and on solubility, never on pass/fail
Relocate the human's attention to where design judgment lives — setting the cradle (the efferent call) and judging solubility — and let the AI generate straight through between. Never gate on red/green. The human owns the call and the solubility verdict; the AI owns generation. → rule `…18-36-39Z-tdd-human-in-the-loop.md`, design doc `agent/design/2026-06-20-human-in-the-loop-tdd-design.md`.

### The pivot: "the definition of the gates is essential"
The gates *are* the essence; some are still missing and unclear. The move: recover the **discriminator** that detects gates, then discover the rest together.

### The gate discriminator + discovery method
A gate = **intrinsic judgment ∩ asymmetry** (AND). Discovery is a pincer: analytical (decompose a feature, classify each decision) + empirical (watch the seams — every human intervention is an un-formalized gate firing). Discovery is itself gated (AI proposes candidates, human judges) — same shape as the thing it designs. Candidate gates: decomposition, intent/correctness, outcome-set, concern, naming. → observation `…18-48-32Z-tdd-gate-discriminator.md`.

### Settling the discriminator
"Settled" = run blind, it produces the verdicts the human would, across corpus + adversarial cases, zero unreconciled disagreements; then it promotes to a rule. Procedure: four passes (calibrate knowns → specificity on non-gates → reconcile candidates → stress the boundary). Two soft spots: **A** make judgment-bearing *intrinsic* (not capability-relative, which would shrink the gate set as models improve); **B** split affordability out of gate-ness. → observation `…18-55-29Z-settling-the-gate-discriminator.md`.

### Decision + passes 1–2
**Soft-spot A ratified:** judgment-bearing is now intrinsic (*underdetermined by the code and tests; needs intent/taste only the human holds*). **B provisionally adopted** (affordability split out), not yet ratified. The discriminator was rewritten; **both legs are now capability-independent**.
- **Pass 1 (calibrate knowns): green** — efferent call and solubility both pass.
- **Pass 2 (specificity): green** — explaining-variable name, `comment` order, whitespace all rejected.
- **Findings:** AND structure validated (variable naming is judgment-ish but rejected via asymmetry); asymmetry is the sharper specificity blade; the "naming" candidate partly resolves — asymmetry splits interface/contract names (gate-ward) from test-local names (not a gate).
- **Remaining:** pass 3 (reconcile candidates), pass 4 (boundary stress), ratify soft-spot B.

### Plain-language statement of the discriminator
Restated the discriminator in plainest terms (added to the gate-discriminator observation as "In plain language"): a gate is where you stop and put the decision to the human; it is a gate only if **(1)** the answer isn't in the code or tests — it takes human taste/intent (*intrinsic*) **and (2)** getting it wrong spreads and locks in (*asymmetric*). Clarified **intrinsic**: the answer isn't present in the artifacts, only in the human's head — anchored to the artifacts, not the model, so it doesn't shrink as models improve (vs. the retired "the AI would guess" phrasing). Examples: naming a local variable → not a gate (derivable + local); the unit's solubility → a gate (unwritten + sets downstream).

### Terminology: "subtle" and "load-bearing" replace "intrinsic" and "asymmetric"
The dense terms ("intrinsic", "asymmetric", "capability-independent") read as academic and risk a preachy, Uncle-Bob-ish tone for an engineering audience the work is ultimately meant to reach. Reframed the two legs in plainer, conveyable language, meaning unchanged: **subtle** (vs **crude**) replaces *intrinsic*, drawing on Bellware's subtle/crude-knowledge distinction — subtle = design principles/qualities, invisible but foundational; crude = tools/patterns, visible/derivable. **Load-bearing** replaces *asymmetric* — other work rests on the choice, so a wrong one spreads and sticks. Dropped "capability-independent" as a coined term and said it plainly instead (subtlety is about the design, not the model, so gates don't shrink as models improve). Reference: https://madabout.software/articles/subtle-knowledge-crude-knowledge/. → updated the gate-discriminator observation.

### Naming the concept: converged on "deliberation"; "load-bearing" ratified
Confirmed **load-bearing** for the second leg. Searched for a more soluble replacement for **discriminator** (correct, but academic and carrying ML/GAN baggage). Walked candidates: *inflection* (turning-point geometry; math + management-speak baggage), *reflection* (right essence but collides with the Reflection programming concept), *consideration/pause/juncture*, and landed on **deliberation** — clean (no programming collision, not buzzwordy), and it sharpens the framework through its double sense: *deliberate* = carefully weighed AND intentional, the opposite of the AI's averaging. Organizing pair: **the AI generates; the human deliberates.** Consequences: "discriminator" retires (the test becomes "does this call for deliberation?"); subtle + load-bearing become the *reasons* a point calls for deliberation. Resolved: **gate stays** (the mechanism — where the loop stops). **hinge** kept as the decision the design turns on (subtle + load-bearing), which also relieves "gate" of its old double-duty (decision vs. stop).

### Settled lexicon
- **hinge** — the decision the design turns on; subtle + load-bearing. The object.
- **gate** — where the loop stops to hand a hinge to the human. The mechanism.
- **deliberation** — the human's act at the gate. (The AI generates; the human deliberates.)
- **subtle** / **crude** and **load-bearing** — the two marks that make a decision a hinge.
- **"discriminator" retires** — the test it named is just "is this a hinge?", and the answer routes to a gate.

One-liner: *the AI generates straight through, and gates at the hinges — the subtle, load-bearing decisions — for the human to deliberate.*

Propagated (2026-06-20): rewrote both observations and renamed them → `…-tdd-hinge-marks-and-discovery.md` and `…-settling-the-hinge-marks.md`; reworked the human-in-the-loop rule and the design doc around hinge / gate / deliberation; "discriminator" retired.

### Two more terms: "mechanical" adopted; "interjection" considered and declined
- **Mechanical / mechanics** replaces "non-hinge" — a positive, non-negated term that scales to bulk generated code, with no programming collision. (Rejected for overloading: given→BDD, routine→subroutine, default, fixture.) Hinges take deliberation; everything else is mechanical.
- **Interjection** considered for the human's act (it conveys the interrupt) but **declined** in favor of keeping **deliberation**: the interrupt is already the **gate**'s job (interjection would duplicate it); "interjection" grammatically means a brief exclamation (underselling considered judgment); and it loses deliberation's double sense (carefully weighed + intentional — the opposite of the AI's averaging).

### Vocabulary frozen; glossary promoted to a rule
Froze **mechanical** (dropped the "for now"). Promoted the canonical glossary from an observation to a binding **terminology rule**: `agent/rules/2026-06-20T21-12-34Z-tdd-lexicon.md` (alongside the efferent-not-caller rule). The observation `…19-46-18Z-tdd-lexicon.md` is kept as a tombstone pointing to the rule. The *framework* (which hinges exist, whether the two marks are final) remains under discovery; only the vocabulary is frozen.

## 2026-06-26 — The prior question: *whether* to interject

### The "whether" is upstream of the "where"
Raised: the whole hinge/gate apparatus answers *where* the human interjects and *how*, but presupposes *that* the human interjects — and it remains possible to let the AI proceed **entirely unscrutinized** (set the call, judge solubility, resolve hinges, generate straight through, no gate). So the live question is *whether* to interject, upstream of *where*.

### Resolution: it's a second dial, not the hinge marks
The hinge marks (subtle + load-bearing) locate **leverage, not necessity** — where judgment pays off *given* the human is in the loop, not that the human must be. This splits the framework into two dials, rightly separated: **(1) hinge-ness** — subtle ∧ load-bearing, a fact about the *design*, capability-independent (soft-spot A's whole point); says *where the risk is*. **(2) gate-policy** — given a hinge, do we actually stop? Capability- and cost-**dependent**; says *whether to pay to cover it*. Capability was pushed out of dial 1 deliberately; it re-enters at dial 2, legitimately. The whether-question is dial 2, and it can be turned to "don't interject."

### The tension, the escape hatch, the floor
- **Tension:** "subtle" = *absent from artifacts, only in the human*. At a real hinge, unscrutinized = the AI can only **average**, which the thesis forbids — an argument to interject *at hinges specifically*, threatening to collapse "whether" back into "where."
- **Escape hatch (already in the thesis):** "condition on this project's high-solubility code" pre-loads the subtle knowledge into the AI; to that degree the gate is overhead. **Better conditioning → fewer gates.** "Unscrutinized" = the framework with gate-policy dialed to *trust*, valid to the degree conditioning absorbed the subtlety. Terminal aim: drive gates toward zero by transferring subtle knowledge into conditioning.
- **Floor (recursion):** "whether to interject" is itself the most load-bearing hinge, so it must be deliberated at least once by the human to set the policy. The human can delegate the loop but not the decision to delegate without first making it. One irreducible interjection.

### Consequence for soft-spot B
This reframes affordability: it is not a downstream *filter* on hinge-ness — it **is** the gate-policy axis (dial 2). → observation `agent/observations/2026-06-26T16-40-50Z-whether-to-interject.md`.

### The inquiry reframed: what is the human's *role*?
Stepped back: the hinge/gate/deliberation machinery is **instrumentation** (where/when the human acts); it never names *what the human is for*. We had mapped the controls without naming the role. The real object of discovery is **the human's role in AI TDD** — and the hinge framework is a hypothesis about it, not the role itself.

**Proposal: the human is the source and guarantor of the standard of "good."** Every thread converges on one ingredient only the human supplies — *subtle* = absent from artifacts/only in the person; "don't regress to the mean" needs an external referent for above-the-mean; solubility can't be certified by the green bar. The AI can generate but cannot *originate* the standard (it's subtle by definition), and its native move without the standard is to average = definitionally not-good. So the machinery is recast: hinges/gates = the points where the standard must be injected; deliberation = one mode of injecting it; cradle + solubility = the human imposing the shape standard and the quality standard.

**The role is a portfolio, split essential vs transitional:** *Originator* (holds what "good" means — essential, can't be delegated, it's what's delegated *from*) and *Boundary-setter* (what counts as a hinge; gate-policy — essential; the irreducible floor) vs *Decider* (in-loop hinge resolution — transitional) and *Judge/ratifier* (validates against the standard — transitional, slower). *Conditioner* (transfers the standard into the AI ahead of the loop) is **the lever** that converts transitional load into the AI. Ties to the two dials: gate-policy is transitional, conditioning moves it → AI TDD is a **migration** of the human from per-instance deciding toward originating + conditioning + boundary-setting; the human becomes the *author of the standard the loop runs on*.

**Sharpened research question:** no longer "where are the hinges" but **how the standard transfers from human into AI conditioning, and what residue never transfers** — the residue *is* the permanent human role (candidates: originator, boundary-setter). → observation `agent/observations/2026-06-26T16-54-59Z-the-humans-role.md`.

### Sharpening: the human is the above-the-mean element
"Don't regress to the mean" requires an external referent for *above* the mean — with teeth. The Human/AI configuration must contain an element above average, above the AI's natural inclination to average; **the human is that element.** The AI's gravity is "good enough" (the mean); the human is inclined to "better" — and *better than the AI has encountered*. Irreducible because averaging only reproduces the *encountered* distribution; "better than encountered" is outside it, so it can't be recovered from corpus/artifacts — only **introduced**, by the human. So the human's role isn't *access to* a standard but *being the super-mean element*; this is the mechanism beneath **Originator** and why delegating it collapses the configuration to the mean. → folded into `2026-06-26T16-54-59Z-the-humans-role.md`.

### Reconciliation: "the portfolio is the role" vs. the single-essence thesis
Asked whether "the portfolio is the role" (no residue once the modes are named — the role is irreducibly plural) competes with "the human is the source/guarantor of the standard" (a single essence). It doesn't; they are two levels. **Essence** answers *why a human is here at all* — the configuration needs an above-the-mean element and only the human can be it (= the Originator mode, irreducible). **Portfolio** answers *what the human does all day* — that essence operationalized as a managed allocation across modes (originator, boundary-setter, conditioner, decider, judge); no single verb for the day-to-day, just the mix. So originating the standard is the irreducible core; the portfolio is its operational form. The other modes are the essence acting through different mechanisms (conditioning transfers it ahead, deciding injects in-loop, judging defends against drift, boundary-setting places where it must enter). The "moving part": as conditioning succeeds, weight migrates out of transitional modes into the AI, concentrating the human on essence (originating) + standard-about-the-standard (boundary-setting). → folded into `2026-06-26T16-54-59Z-the-humans-role.md`.

### Terminology: items are *modes*, not roles; "purpose" replaces "essence"
Two naming decisions on the human-role material. **(1) Items renamed.** "Portfolio" had been standing in for "portfolio of roles," overloading **role** (the human's whole position vs. each item). Resolved: **role** is reserved for the whole; the items are **modes of the one role** (originator, boundary-setter, conditioner, decider, judge), not separate roles — so no collective noun is needed and the "set of roles" question dissolves. The allocation insight "portfolio" carried (weighting shifts off transitional modes as conditioning succeeds) is kept, expressed without a special noun. Item word (**mode** vs **capacity**) and role-level wording deferred ("decide later"); not frozen. **(2) "Essence" → "purpose"** throughout the reconciliation — the level it names answers *why a human is here at all*, which is purpose. → updated `2026-06-26T16-54-59Z-the-humans-role.md` (the reconciliation and modes sections rewritten).

## 2026-06-26 — Exploring TDD-AI in action (the Name feature) + the gate-forecasting speculation

### Proposed a hypothetical process for Task 3 (Name)
Framed running Task 3 as **build + experiment**: the framework predicts the human interjects only at hinges, so forecast the gates and treat every actual interjection as data (predicted gate confirms; unpredicted gate names an un-formalized hinge; predicted gate needing no interjection = conditioning already absorbed that standard). Grounded in the real idiom (`define_constant.rb`, `Controls::Constant.example`). Turn-by-turn: turn 0 human sets gate-policy (boundary-setter, the floor interjection); turn 1 AI writes only the test file = the proposed efferent call → **GATE 1** (message/shape; *which example* — a top-level example can't distinguish "final segment" from full path, only a nested one can = predicted intent/correctness seam; assert against control-provided expected, not a literal); turn 2 AI generates inside the cradle, runs to confirm; **GATE 2** solubility (+ naming seam: `name` vs `Module#name`); turn 3 commit + log + auto-record. Predicted seams: intent/correctness (fires), outcome-set (maybe — anonymous module → nil name?), naming (maybe), decomposition (predicted *not* to fire — negative test). Observable sought: **ratify vs correct** per gate — ratify = conditioning carried it; correct = standard not yet conditioned in → a conditioning candidate.

### Speculation: gate forecasting could be a key to the process
The AI **predicts the gates and announces them to the human before acting** — the gate becomes *signaled by the AI ahead* rather than *tripped by the human after*. Why it might matter: flips discovery reactive→proactive; counters unscrutinized proceeding (silent averaging → explicit handoff); is the boundary-setter recursion run inline, AI-first (AI proposes candidate hinges, human ratifies); a conditioning-transfer channel (ratify confirms the AI's hinge-map, override calibrates it — migrates boundary-setting to the AI); economizes the human's scarce above-the-mean attention (a map of where to deliberate = the practical form of the gate-policy dial). Where it breaks: **structural blind spot** (the AI can't forecast what it can't see, and hinges are *absent from the artifacts* by definition — weakest exactly where it matters; supplements, not replaces, vigilance); **false alarms** (over-forecasting re-imposes the cost and trains rubber-stamping — needs few misses and few false alarms); **mean-bias** ("here's the call I'm about to make, ratify?" pulls the human toward the AI's mean-drawn answer and contaminates the ratify-vs-correct signal — present alternatives, not a fait accompli). Net: a supplement that focuses attention + a calibration channel for transferring the hinge-map. Held as early speculation. Test it on the Name run (did forecasting help or exert mean-bias?). → observation `agent/observations/2026-06-26T17-17-05Z-gate-forecasting.md`. *(Terminology updated 2026-06-26: "mean-bias" replaces "anchoring"; "miss / false alarm" replaces "recall / precision".)*

### Theorizing the *form* of a good forecast — three laws
Pushed on the failure modes; a shape fell out. **(1) Mean-bias.** A naive forecast ("here's my call — ratify?") makes the human evaluate-the-proposal (judge mode) instead of originate-the-best (originator mode); judge mode is downstream of the AI's mean-drawn proposal, so the forecast exerts **mean-bias** — pulling the above-the-mean element *down toward the mean* and demoting the human from the irreducible mode to a transitional one. The anti-averaging mechanism induces averaging *in the human*. → **Law 1: provoke origination, not mean-bias.** **(2) Calibration is asymmetric.** A **false alarm** (flagging a mechanical decision) is cheap + instructive; a **miss** (failing to flag a real hinge) is expensive + invisible ⇒ lean toward flagging — but leaning too hard habituates rubber-stamping, restoring the misses. **Resolution: forecast the *partition*, not the gates** — tier 1 gates / tier 2 suspected / tier 3 mechanical-about-to-decide-silently. The dangerous miss lives in the *proceed pile* (a hinge the AI can't see is one it filed as mechanical), so exposing tier 3 is where misses are caught; tiering protects the tier-1 attention channel. A partition is about *which decisions need you*, not the answers — keeps the human in boundary-setter/originator, not judge (so it also satisfies Law 1). → **Law 2: expose the proceed-pile.** **(3) Blind spot bounded, not solved:** the partition can't find the unseen hinge, but converts unknown-unknown → known region to audit (tier 3) — "points at where the unseen lives"; catch-the-seam vigilance stays irreducible. **Meta-payoff:** every human *re-filing* (tier 3↔tier 1) is labeled hinge-map data = conditioner output → forecasting *is* the mechanism that migrates boundary-setting to the AI; the forecast generates its own training signal. **Catch:** exhaustive partitions don't scale → **Law 3: keep it step-local** (altitude of the current cradle). Compressed: a good forecast is *a step-local, confidence-tiered decision partition that leans toward flagging, presented to provoke origination not mean-bias — its re-filings the conditioning signal that migrates the hinge-map to the AI.* Still speculation, untested. (Terms: "mean-bias" not "anchoring"; "miss / false alarm" not "recall / precision".) → folded into `2026-06-26T17-17-05Z-gate-forecasting.md`.

### Is "mean-bias" a lowering, or just regression to the mean? (term under review)
Caught that "mean-bias" names a *direction* (toward the mean), not a *harm*: bias toward the mean is a **lowering** only if you start *above* the mean; for a below-the-mean design the same pull is a *raising* — an improvement. So the term over-claims badness, smuggling in the above-the-mean premise. **Rescue: scope to hinges.** Law 1 governs only how a gate (hinge) is presented, and on a hinge the human is above the mean *by definition* (a hinge is subtle → the AI can only average → its proposal can't hold the subtle knowledge → the human can do better). So at a hinge, bias toward the AI's proposal is always a lowering; the below-mean case only occurs on *mechanical* decisions, where there's no gate, so Law 1 never applies. The invariant Law 1 forbids is the **lowering** (human pulled to produce worse than their unbiased best); "toward the mean" is just what lowering looks like above the mean — which at a hinge you always are. **Resolved:** keep **mean-bias** for the *mechanism* (the forecast pulling the human toward the AI's mean-drawn answer); use **lowering** for its specific *effect at a hinge* (the human producing worse than their unbiased best). Mean-bias is the cause; lowering is what it does at a gate. Law 1 stays *provoke origination, not mean-bias*; the harm it prevents is the lowering. → folded into `2026-06-26T17-17-05Z-gate-forecasting.md`.

### Two limits on Law 1, recorded as open questions
**A — options the AI can't produce:** "present the alternatives" (the mean-bias mitigation) doesn't escape the lowering, because the list is the AI's own (drawn from the average) and showing it makes the person *pick from it* instead of producing their own. The best option may not be on it — and the option that mattered is often one only the person could produce. **Survives even as the AI improves** (it's about options the AI can never produce, not the quality of its average) → ties to what stays with the person permanently. **B — stage-dependence:** Law 1 forbids mean-bias *because the proposal is mean-drawn* — expires as conditioning lifts the proposal above the mean; then provoke-origination becomes wasteful and propose-for-ratification is fine (judge mode legit on an above-mean proposal). Danger: switching too early reintroduces lowering; the switch is a hinge needing a conditioning-confidence measure. **Dissolves with conditioning** (by design), if the switch is governed right.

### Developing the conditioning-confidence measure
The governing measure for B. **Ratify-vs-correct already is the confidence signal** (ratify = standard transferred for that class; correct = not yet) — the transfer instrument doubles as the measurement, same data stream. **Circularity trap:** under propose-for-ratification the human is mean-biased toward ratifying, inflating the rate — the switch corrupts its own justifying signal. So measure confidence **only under provoke-origination**, via a **blind-audit channel**: AI commits its proposal privately, human originates **free-form** (open-ended, not multiple-choice — else limit A poisons it), reveal and score the match; sample it (costs attention). Needs: **per-hinge-class** granularity; **severity weighting** (gate the switch on high agreement AND low worst-case load-bearing-ness — catastrophic-miss classes never leave provoke-origination); **recency/drift, no permanent graduation** (standard keeps rising, classes can regress); **the switch is a hinge — measure recommends, human ratifies** (automating it = the proceed-unscrutinized trap). **Payoff:** one human act both transfers the standard and measures the transfer; the migration self-paces. **Two permanent taxes:** blind-audit sampling (confidence is never free) and seam vigilance (confidence can't touch the unseen hinge — limit A). → folded into `2026-06-26T17-17-05Z-gate-forecasting.md`.

### Can the AI know its conditioning level? Can it learn what the human is about to think?
**Q1 (self-knowledge):** **empirically yes** — the per-class ratify/correct record *is* the measure, and it's in the artifacts the AI can read. **Introspectively no** — *subtle* = absent from the AI, so feeling the gap would require the gap's content to be inside it (contradiction). Worse, the **fluency trap**: the AI's own confidence is anti-correlated with hinge-awareness (averaging is most fluent on the mean answer), so confident-and-wrong feels safe → trust the record, not the feeling. And the record **lags** (past-tense; standard rises, conditioning decays) → self-knowledge is always one audit behind. **Q2 (other-knowledge):** predicting the human's origination *is* conditioning's success condition. Learnable: the human's *expressed* taste on *seen* patterns (recurring hinges). Not learnable: *novel* origination on new ground (in no artifact yet). **The gap is permanent because prediction relocates the frontier** — once the AI absorbs the standard, "the mean" includes it, and the human's role is to exceed the now-AI-inclusive mean; the AI converges on where the human *was*, the human stays where the AI isn't yet. **Kicker:** over-confident prediction → propose-for-ratification → stops eliciting origination → human (mean-biased to ratify) stops producing above-mean → **the standard freezes** at the level reached when provocation stopped. So the **provoke-origination channel must persist forever** — not as a fallback but as the *engine that keeps the standard rising*; remove it and you collapse the ceiling to the floor. **Reframes the migration:** not "human role → zero," but "human role migrates from recurring hinges (transferable) to the frontier (never transferable, because the AI getting stronger moves the line)." The residue is a *moving frontier the human's function defines*. → observation `agent/observations/2026-06-26T18-05-02Z-ai-self-knowledge-and-predicting-the-human.md`.

### Developing limit A (options the AI can't produce)
*(De-jargoned: replaced the "A1/A2" codes; dropped the "menu" metaphor and "recoverable/irreducible" per the no-metaphor naming principle.)* **The structure is a three-rung ladder:** **(1) Reached** — the AI reached it (produced + surfaced), on the list; **(2) Missed** — the AI could reach it but missed this pass (*fixable*); **(3) Unreachable** — only the person can produce it, the AI never reaches it (*not fixable*). Looks like a 2×2 (reachable/not × surfaced/not) but the 4th cell is **impossible** — reached presupposes reachable, so unreachable options are always off the list; collapses to three rungs. Limit A lives in rungs 2–3. **Missed (rung 2) is fixable** by running several independent lists from different angles (coverage rises); does nothing for the unreachable rung. **Unreachable (rung 3) can't be produced, only kept from suppressing the person:** the harm is that *showing a list makes the person pick from it instead of producing their own* (a list invites selection, a blank start invites origination) — so an explicit list is worse than none here; mean-bias one level up, a Law-1 violation at the option-list level. Remedy = **answer first, compare second** (the person produces before seeing the AI's list; the list is shown after, as a cross-check) — *structurally identical to the conditioning-confidence blind audit*, so **the defense against unreachable options and the confidence measure are one instrument**. **Sliver of self-knowledge:** the AI can't see the missing content but can tell when its list is *all of one kind* (every option a variation on one approach) → a sign it's stuck and missing categories → push for fresh; limit: misses the varied-but-all-below case. **Residuals:** the list still tempts the person to defer (keep their first answer privileged; frame the list as "more candidates, not the full set"); producing-first is costly → reserve for decisions that matter (same dial as B). **Unification:** the option the AI can't produce = the decision it can't recognize as needing the person (forecasting blind spot) = the answer it can't predict (prediction frontier) — *the same single thing seen three ways: what only the person can produce*. A doesn't close; it gets **handed to the person**. → folded into `2026-06-26T17-17-05Z-gate-forecasting.md`.
