# Return constant names as Strings; accept either String or Symbol as input wherever the delegated Ruby method does

The library is **liberal in what it accepts and uniform in what it returns** (the robustness principle), applied to constant names:

**Outputs — always String.** Any constant name the library hands back is a `String`, anchored on `Module#name` (itself a String). Where a name is derived from a Ruby method that returns Symbols — chiefly `Module#constants` — normalize it to a String on the way out (`raw_constant.constants(...).map(&:to_s)`). The caller never has to branch on whether a returned name is a String or a Symbol; it is always a String.

**Inputs — accept both, by delegation, never by coercion.** A method that takes a constant name accepts **either a String or a Symbol whenever the underlying Ruby method it delegates to accepts both** — `const_set`, `const_get`, `const_defined?`, and the like all do. Achieve this by *passing the name straight through* to that method; do **not** add `to_sym`/`to_s` coercion or type-validation that narrows what Ruby would already accept. If a method delegates to a Ruby method that accepts only one type, match that one type — the permissiveness tracks the delegate, it is not a blanket guarantee.

**Why:** Ruby is mixed — `Module#name` is a String, `Module#constants`/`Method#name` are Symbols, and `const_*` accept both. Mirroring that split on the *output* side would force callers to branch on type; normalizing outputs to String removes that burden and makes returns predictable. On the *input* side, the `const_*` methods are already permissive, so narrowing them would be the library imposing friction Ruby doesn't — accepting whatever the delegate accepts keeps the library frictionless and honest about where flexibility actually comes from.

**How to apply:** When a method returns a constant name (or list of names), return Strings — `.map(&:to_s)` over any `Module#constants`-derived result. When a method takes a constant name, delegate it unmodified to the `const_*` (or equivalent) call and let Ruby's own polymorphism carry it; add no coercion or type check. This **supersedes** the earlier decision that `#constant_names` would stay Symbols (mirroring `Module#constants`) — `constant_names` now returns Strings; see log `…2026-06-26T21-04-46Z-name-returns-string.md` (amended) and the convention's own log `…2026-06-27T23-06-29Z-string-outputs-permissive-inputs.md`. Note: a name the library *returns* is a String, but a name Ruby itself returns in passing (e.g. a `Module#constants` Symbol shown in a test `comment`) is left as Ruby gives it.
